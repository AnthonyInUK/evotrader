# 我让四个 AI 分析师开了个投资会议，然后它们吵起来了

> 本文适合对 AI agent、量化交易、或多智能体系统感兴趣的读者。不需要有金融背景。

---

## 先说一个场景

想象你是基金经理，开会前收到了四份研究报告：

- 基本面研究员：「茅台 ROE 34%，净利率 52%，强烈看多，信心 95%」
- 技术分析师：「均线多头排列，MACD 金叉，动量强劲，信心 85%」
- 市场情绪分析师：「北向资金持续流入，外资最重仓的 A 股标的，看多」
- 宏观分析师：「消费复苏逻辑成立，无明显政策风险，中性偏多」

你会怎么决策？

大多数人会说：四个人都看多，买。

但这里有个问题——四份报告回答的都是同一个问题：「茅台这家公司好不好？」。没有一份报告回答了另一个问题：「我现在手里已经有这么多茅台了，还应该继续加仓吗？」

这就是我做这个项目想解决的问题。

---

## 系统是怎么设计的

我用 [AgentScope](https://github.com/modelscope/agentscope)（ModelScope 的多 agent 框架）搭了一套 A 股多 agent 量化交易系统，叫 EvoTraders。

结构分四层：

**四个 Analyst Agent**，各司其职：
- `fundamentals_analyst`：爬取财务数据，看 ROE、净利率、增长率
- `technical_analyst`：计算均线、MACD、RSI，看价格形态
- `sentiment_analyst`：分析资金流向和市场情绪
- `macro_analyst`：判断宏观环境和政策风险

**Risk Agent**：读完四份 analyst 报告，给出风险评估建议（软约束）。

**Portfolio Manager Agent**：综合所有信息，输出最终交易指令——买多少、卖多少、或者按兵不动。

**RiskGuard（规则引擎）**：在 PM 决策之后、实际执行之前强制过滤。仓位超 30% 就裁剪，当日亏损超 5% 就熔断，逻辑写死在代码里，LLM 说了不算。

---

## 为什么选 AgentScope，而不是 LangChain

这是我被问得最多的问题，说一下选型理由。

LangChain 的 multi-agent 是通过 State 对象传递信息的——Agent A 把结果写进 State，Agent B 从 State 里读。信息是结构化的，流向是单向的。

AgentScope 的核心原语是 **MsgHub**——一个共享的对话线程。所有 agent 都在同一个「会议室」里，可以看到彼此刚才说了什么，然后做出回应。

这个区别在我的场景里很关键。量化投资里，分析师之间的分歧往往比共识更有价值——技术面说涨，但基本面说贵，这两个信号的张力才是真正需要 PM 去权衡的东西。如果信息通过 State 传递，这种张力在传递过程中就被压平了。MsgHub 让每个 agent 的判断保持独立，PM 在充分了解分歧的情况下做决策。

用一句话概括：**LangChain 适合流水线，AgentScope 适合开会。**

## 2024-02-23 发生了什么

这一天是三个月回测的净值最高点，账户资产 ¥521,741，累计收益 +4.35%。

四个 analyst 全部看多茅台。以下是系统 reasoning log 里的真实输出：

**fundamentals_analyst：**
> "ROE 34.2%，净利率 52.5%，营收增速 18%。定价权 + 极低债务 + 稳定现金流。A 股净利率最高的消费股没有之一。信号：**Bullish，信心 95%**。"

**technical_analyst：**
> "收盘价 1607.77，站在 SMA20（1549）和 SMA60（1550）上方。MACD 金叉，20 日动量 +6.9%。价格走势稳健，无超买迹象（RSI 63.4）。信号：**Bullish，信心 85%**。"

**sentiment_analyst：**
> "茅台是北向资金通过沪港通持仓最重的 A 股标的。RSI 63.4 反映的是机构有序建仓，而非散户追涨。信号：**Bullish**。"

然后 Portfolio Manager 读完这三份报告，输出了：

```
600519.SH: SHORT（减仓）100股  信心 90%

理由：当前仓位占比 30.9%，超过 30% 集中度上限。
DCF 估值显示当前股价比内在价值高出 32%。
好公司，但价格贵了。T+1 解锁确认今日可卖。减仓。
```

**PM 做了和所有 analyst 相反的决定。**

---

## 为什么这个结果是合理的

两个逻辑：

**analyst 和 PM 回答的不是同一个问题。**

analyst 回答：「茅台这家公司好不好？」——好，非常好。

PM 回答：「我现在应该持有多少茅台？」——已经 30.9% 了，再加就是把三分之一的钱押在一只股票上。

即使公司很好，过度集中本身就是风险。

**好公司不等于好价格。**

DCF（现金流折现）是一种估值方法，逻辑很简单：把公司未来几年预计赚到的钱折算成今天值多少。结果显示，茅台当前股价比它「理论值」高了 32%——市场已经提前把未来几年的增长都定价进去了。analyst 看的是公司质地，PM 看的是现在买贵不贵，两个问题可以有完全不同的答案。

---

## 决策链路怎么追溯

每个交易日结束后，系统把所有 agent 的完整推理轨迹写入日志文件：

```
logs/daily/2024-02-23_reasoning.txt
```

里面有：
- 每个 analyst 调用了哪些工具、拿到了什么数据、得出了什么结论
- Conference 讨论阶段各 agent 之间的完整对话
- PM 最终决策的结构化 JSON 和完整 reasoning

2 月 23 日的日志是 1062 行。当天 PM reasoning 写了 1800 字，把减仓逻辑说得非常清楚。

这不是为了存档——是为了在某天系统亏钱的时候，能精确找到是哪个 agent 的判断出了问题，而不是对着一个黑盒发呆。

## 三个月回测结果

回测区间：2024-01-02 → 2024-03-28，61 个 A 股交易日  
标的：600519.SH（贵州茅台）+ 601398.SH（工商银行）  
初始资金：¥500,000

|  | Agent 策略 | 买入持有基准 |
|--|--|--|
| 最终资产 | ¥511,611 | ¥502,516 |
| 总收益率 | **+2.32%** | +0.50% |
| 超额收益 | **+1.82%** | — |
| 最大回撤 | 1.15% | — |
| Sharpe | 0.87 | — |

Sharpe 衡量的是「每承担一份风险，赚到多少收益」。高于 1 是优秀，0.87 说明没有靠过度冒险赚钱，收益是稳的。

2 月 23 日减仓之后，净值确实在 3 月开始慢慢回落。PM 那天的判断事后来看是对的。

## 几个没解决的问题

**数据有个作弊的地方。** 财报数据用的是报告期，但现实里年报通常要到次年 3 月底才披露。回测里 1 月份就用上了这些数字，会让基本面 analyst 看起来比实际聪明。这个问题叫「前视偏差」，量化回测里最常见的坑，我目前没有修。

**成交是理想状态。** 每次下单都按开盘价全部成交。真实市场里大单会影响价格，实际成交价会更差，这叫滑点，没有模拟。

**agent 每天失忆。** 每个交易日开始前短期记忆清空，agent 看不到自己昨天的判断，也没办法意识到「我这周已经连续看错三天了」。AgentScope 框架支持长期记忆（ReMeTaskLongTermMemory + 向量检索），我正在接入，准备做个有记忆和没记忆的对比实验，会再写一篇。

---

## 最后

做这个项目最有意思的地方不是收益率，是看 agent 之间产生真实分歧的过程。

analyst 全部看多，PM 选择减仓——这个张力在单 agent 系统里不会出现，因为所有信息被打包进一个 prompt，LLM 会自动做平均，分歧被掩盖了。多 agent 的设计让每个维度的声音保持独立，然后强迫一个更高层次的 agent 在分歧中做决策。

这才是「让分歧可见」这个设计原则想要的效果。

项目基于 [AgentScope](https://github.com/modelscope/agentscope)，LLM 用的是阿里云 DashScope qwen-plus，数据来源 akshare。源码整理后会放出来。
