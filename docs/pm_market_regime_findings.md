# PM 市场环境实验复盘

这份文档记录 8 股票池 PM 规则在不同市场环境下的实验结论。

核心目的不是证明策略已经能赚钱，而是回答一个更工程化的问题：

```text
当前 PM 规则在弱市、反弹、小涨震荡三种环境下，行为是否一致、是否过度保守、是否需要改规则。
```

## 为什么要做这组三环境实验

一月完整回测跑赢之后，不能直接得出“PM 规则有效”的结论。

原因是：

```text
一月是弱市。
弱市里，高现金天然容易跑赢满仓基准。
如果只看一月，我们可能把“少亏”误判成“会选股”。
```

所以实验设计必须拆成三个市场环境：

| 环境 | 目的 | 想验证的问题 |
|---|---|---|
| 弱市 | 验证防守能力 | PM 高现金是否能控制回撤、避免跟跌 |
| 反弹 | 验证进攻能力 | 市场转好时 PM 是否敢提高仓位 |
| 震荡/小涨 | 验证平衡能力 | PM 是否既不过度交易，也不会现金过高 |

这就是为什么后面没有继续在一月里微调，而是选择 2 月和 5 月各跑 5 天 smoke。

## 实验设置

固定条件：

```text
股票池：8 只 A 股
分析师：不改
模型：不改
数据工具：不改
固定指标：不改
PM 主规则：不改
```

唯一变化：

```text
市场环境 / 时间窗口
```

这样实验结果才可以归因到 PM 对不同市场环境的适应能力，而不是混在分析师、模型、数据工具变化里。

## 实验结果总览

| 实验 | 窗口 | 市场环境 | Agent 收益 | 基准收益 | 超额收益 | 结论 |
|---|---:|---|---:|---:|---:|---|
| `backtest_202401_pool8_level1_behavior` | 2024-01 | 弱市 | +0.81% | -3.80% | +4.61% | 跑赢，主要靠现金防守 |
| `backtest_202402_rebound_pool8_smoke` | 2024-02-20 ~ 02-23 | 反弹 | +1.89% | +1.98% | -0.09% | 小幅跑输，仓位略保守 |
| `backtest_202405_sideways_pool8_smoke` | 2024-05-13 ~ 05-17 | 震荡/小涨 | +0.17% | +1.06% | -0.89% | 跑输，现金拖累明显 |

注意：2 月和 5 月并不是 PM 不赚钱，而是上涨环境里没有跑过基准。

## 一月弱市：为什么跑赢

一月归因：

```text
Agent 最终资产：¥504,051.56
等权基准：¥481,019.76
超额：¥23,031.80
```

拆开看：

```text
现金/风控贡献：约 +¥18,980
主动股票贡献：约 +¥4,052
```

个股贡献：

| Ticker | 贡献 |
|---|---:|
| 000333.SZ 美的 | +¥6,580 |
| 000858.SZ 五粮液 | -¥1,344 |
| 600276.SH 恒瑞 | -¥484 |
| 601398.SH 工行 | -¥480 |

结论：

```text
一月不是“PM 很会进攻”，而是“PM 防守有效”。
它没有满仓跟跌，同时选中了美的这个相对强标的。
```

这说明当前 PM 规则在弱市有价值：

```text
没有强信号时保留现金
不强行买满股票池
遇到不确定性时少犯错
```

但它不能证明 PM 在上涨月份也适合。

## 二月反弹：为什么小幅跑输

二月反弹 smoke 结果：

```text
Agent：+1.89%
等权基准：+1.98%
超额：-0.09%
```

归因：

```text
现金拖累：约 -¥9,895
主动股票贡献：约 +¥9,427
```

个股贡献：

| Ticker | 贡献 |
|---|---:|
| 000858.SZ 五粮液 | +¥5,618 |
| 601398.SH 工行 | +¥3,888 |
| 600276.SH 恒瑞 | +¥585 |
| 000333.SZ 美的 | -¥429 |
| 300750.SZ 宁德时代 | -¥77 |

结论：

```text
PM 并不是没选对股票。
它买的五粮液、工行、恒瑞整体赚钱。
真正拖累是现金太多，以及对强势仓位偏早止盈。
```

关键行为：

```text
PM 会在反弹里提高仓位，不是完全保守。
但它仍然倾向于保留较高现金，并提前减掉盈利仓。
```

这说明当前规则的问题不是“不会买”，而是：

```text
市场转强时，目标仓位没有同步上调。
```

## 五月震荡/小涨：为什么跑输

五月 smoke 结果：

```text
Agent：+0.17%
等权基准：+1.06%
超额：-0.89%
```

归因：

```text
现金拖累：约 -¥5,317
主动股票贡献：约 +¥850
```

个股贡献：

| Ticker | 贡献 |
|---|---:|
| 000858.SZ 五粮液 | +¥2,996 |
| 000333.SZ 美的 | -¥1,482 |
| 600276.SH 恒瑞 | -¥580 |

结论：

```text
PM 在震荡/小涨环境里没有乱交易，这是优点。
但现金仍然过高，导致上涨时跟不上基准。
```

这说明：

```text
PM 的“避免犯错”能力存在。
但当市场不是明显弱市时，它缺少主动提高风险预算的机制。
```

## 当前 PM 画像

基于三组实验，当前 PM 不是简单的“好”或“坏”，而是有清晰风格：

```text
弱市：适合，现金防守有效
反弹：能赚钱，但容易踏空一部分
震荡/小涨：交易克制，但仓位偏低
```

能力：

```text
能控制回撤
能避免满仓买入弱股票池
能做一定选股排序
能在试错后减仓或退出
```

问题：

```text
缺少市场环境识别
缺少目标仓位机制
上涨环境里现金过高
强势盈利仓可能过早止盈
```

## 和实验设计的关系

这组三环境实验验证了一个实验设计原则：

```text
不能只在一个月份调 PM。
必须先判断 PM 在不同市场环境下的行为模式，再决定改哪里。
```

如果只看一月：

```text
我们会以为 PM 很优秀。
```

如果只看二月或五月：

```text
我们会以为 PM 太保守、需要大幅提高仓位。
```

三组一起看，才得到更稳的结论：

```text
PM 的防守逻辑有效，不应该删除。
但它需要市场环境感知层，在反弹/小涨环境中提高目标仓位。
```

所以后续实验不应该盲目改分析师，也不应该直接把 PM 改激进。

正确的下一步是只改一个变量：

```text
增加 PM 市场环境感知 + 目标仓位规则。
```

## 下一版 PM 规则方向

不要把 PM 从“保守”直接改成“激进”。

已实现为 PM 实验开关：

```text
PM_EXPERIMENT_MODE=regime_target_exposure
```

它增加一个上层规则：

```text
先判断市场环境
再决定组合目标仓位
再在股票池中分配仓位
```

建议目标仓位：

| 市场环境 | 目标股票仓位 | 现金 |
|---|---:|---:|
| 弱市 | 40%-60% | 40%-60% |
| 震荡/小涨 | 65%-75% | 25%-35% |
| 明确反弹 | 80%-90% | 10%-20% |

这不是最终参数，只是下一轮实验的候选变量。

实现位置：

```text
backend/agents/portfolio_manager.py
backend/core/pipeline.py
env.template
```

其中 pipeline 会把 `portfolio_exposure` 传给 PM：

```text
cash
positions_value
total_equity
equity_exposure_pct
cash_pct
```

PM 在启用开关后，必须先说明：

```text
当前 regime
当前股票仓位
目标股票仓位
本日交易如何让组合靠近目标仓位
```

### v2 修正：Deployment Discipline

第一次 `regime_target_exposure` smoke 暴露了一个新问题：

```text
PM 能判断 REBOUND，也能说目标仓位 80%；
但实际订单只把股票仓位提高到 60% 左右。
```

这说明 v1 只解决了：

```text
先判断市场环境
```

但没有真正解决：

```text
再在股票池里分配仓位
```

因此 v2 增加部署纪律：

```text
如果 REBOUND 且当前仓位低于目标下沿，必须继续向候选股票部署现金。
如果第一候选因为一手太贵、单票上限、现金上限买不了，必须把预算转给下一梯队。
如果最终仍低于目标仓位，必须列出具体 blocker，不能只说“谨慎”。
```

这个修正的目的不是让 PM 无脑满仓，而是让它把“目标仓位”变成真实订单。

## 下一轮实验设计

实验名建议：

```text
backtest_pm_regime_target_exposure_v1
```

唯一变量：

```text
PM 增加市场环境感知和目标仓位约束。
```

固定条件：

```text
股票池不变
分析师不变
模型不变
数据工具不变
固定指标不变
回测窗口不变
```

先不跑整月，只跑同样的 smoke 窗口：

```text
弱市：2024-01 中 5 个交易日
反弹：2024-02-20 ~ 2024-02-23
震荡/小涨：2024-05-13 ~ 2024-05-17
```

通过标准：

```text
弱市不能显著恶化回撤
反弹月现金不能长期高于 25%-30%
震荡月不能因为提高仓位而频繁乱交易
超额收益改善来自仓位和选股，而不是某一笔偶然交易
```

失败标准：

```text
弱市回撤明显放大
反弹月仍然高现金
震荡月交易次数明显增加但收益没有改善
PM 开始无视风险/估值/集中度规则
```

## 一句话总结

```text
当前 PM 的弱市防守能力已经被一月验证；
二月和五月暴露的问题是上涨环境仓位不足；
下一步应该只增加“市场环境 → 目标仓位”的 PM 规则，
而不是同时改分析师、模型或股票池。
```
